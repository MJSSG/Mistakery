## 一、核心功能模块

### 1. 错题录入

- **简化录入流程**

  - 仅需选择科目和子分类
  - 将完整题目文本粘贴到文本框
  - 一键提交，系统自动解析
- **支持的题目类型**

  - 单选题
  - 多选题
- **科目分类系统**

  - 政治理论（马思、毛思、习思、中特、党史、时政）
  - 常识判断（经济、科技、历史、人文、地理、法律）
  - 言语理解（片段阅读、逻辑填空、语句表达）
  - 判断推理（图形推理、类比推理、定义判断、逻辑判断）
  - 数量关系（简单计算、行程问题、工程问题、经济利润、排列组合、概率、几何）
- **自动解析功能**

  - 自动提取题目内容、选项、答案
  - 智能判断题目类型（单选/多选）
  - 自动推算难度等级（根据全站正确率）
  - 支持时间戳、答题用时、解析等可选字段
- **数据存储**

  - questions 表：存储题目基础信息
  - mistakes 表：存储用户错题记录
  - 重复题目警告模式（复用已有题目ID）
- **用户界面**

  - 图标化科目选择器
  - 动态加载子分类选择器
  - 大文本题目编辑器
  - 预览解析、清空、提交按钮
- **API接口**

  - `POST /api/mistake/parse-and-save`
  - 返回 questionId、mistakeId 和解析后的结构化数据

  **错题格式：**

  从现实角度来说，“好面子”实际上是一种遵守社会游戏规则的表现。如果不把面子事做好，就容易被周围的人________。在很多人看来，根本不必考虑“好面子”背后的原因和目的，只要跟大多数人的行为保持一致，________，就一定不会错，这就是社会心理学中经常提到的“羊群效应”。

  依次填入画横线部分最恰当的一项是：

  **A**反感 亦步亦趋

  **B**排斥 随波逐流

  **C**鄙视 墨守成规

  **D**误解 人云亦云

  我的答案：A

  正确答案：B

  答题用时：**51秒**

  全站正确率：**67%**

  解析

  【思路】

  先从第二空入手，由“只要跟大多数人的行为保持一致”以及“羊群效应”可知，横线处要表达“从众行为”。A项“亦步亦趋”比喻自己没有主张，每件事都效仿或依从别人，跟着人家行事，B项“随波逐流”比喻没有坚定的立场，没有主见，盲目地随着别人行动，均能体现在行为上与别人一致，符合文意，保留。C项“墨守成规”形容死守老规矩，保守固执，不求改进，与文意不符，排除；D项“人云亦云”人家怎么说，自己也跟着怎么说，形容没有主见，而文段强调“行为一致、做”，与文意不符，排除。

  第一空，根据“‘好面子’实际上是一种遵守社会游戏规则”可知，如果没把面子的事做好，就是没有遵守社会游戏规则。A项“反感”指厌恶；不满，B项“排斥”不相容、使离开或不使进入。不遵守社会规则的人是不会被大家接纳的，“反感”程度不够。

  因此，答案为B。

## 三、技术特性

### 9. 错题录入技术实现

- **前端文件结构**（Vue 3 + Vite）

  ```
  src/
  ├── views/MistakeEntry/
  │   ├── MistakeEntry.vue           # 主页面组件
  │   └── styles/                     # 样式文件
  ├── components/MistakeEntry/
  │   ├── CategorySelector.vue       # 科目选择器
  │   ├── QuestionEditor.vue         # 题目编辑器
  │   └── PreviewPanel.vue           # 预览面板
  ├── composables/
  │   ├── useMistakeForm.ts          # 表单逻辑
  │   └── useQuestionParser.ts       # 解析逻辑
  └── services/
      └── mistakeEntry.ts            # API服务
  ```
- **后端文件结构**（NestJS 模块化）

  ```
  src/
  └── modules/
      └── mistake/
          ├── mistake.controller.ts      # 控制器
          ├── mistake.service.ts         # 业务逻辑
          ├── mistake.module.ts          # 模块配置
          ├── dto/
          │   ├── parse-mistake.dto.ts   # 解析DTO
          │   └── create-mistake.dto.ts  # 创建DTO
          ├── entities/
          │   ├── question.entity.ts     # 题目实体
          │   └── mistake.entity.ts      # 错题实体
          └── parsers/
              └── question-parser.service.ts  # 解析服务
  ```
- **核心解析逻辑**

  - 时间戳：`/^##\\s+\\d{4}\\/\\d{1,2}\\/\\d{1,2}\\s+\\d{1,2}:\\d{2}:\\d{2}\\s*/gm`
  - 题目内容：`/^([\\s\\S]*?)(?=\\nA\\s*$)/ms`
  - 选项：`/\\n([A-D])\\s*([\\s\\S]*?)(?=\\n[A-D]\\s*$|\\n我的答案|$)/gm`
  - 用户答案：`/我的答案[：:]\\s*([\\s\\S]*?)(?=\\n|$)/`
  - 正确答案：`/正确答案[：:]\\s*([\\s\\S]*?)(?=\\n|$)/`
  - 答题用时：`/答题用时[：:]\\s*([\\s\\S]*?)(?=\\n|$)/`
  - 全站正确率：`/全站正确率[：:]\\s*([\\s\\S]*?)(?=\\n|$)/`
  - 解析：`/解析[：:]\\s*([\\s\\S]*)$/s`
- **智能判断逻辑**

  - 题目类型：根据答案数量判断单选/多选
  - 难度等级：根据全站正确率判断（易≥70%、中50-69%、难<50%）

- ✅ 错题录入（智能解析、快速录入）

- ✅ 学习统计（数据可视化）
- ✅ 答案解析（举一反三学习）
- ✅ 个人中心（个性化设置）
- ✅ 技术优化（性能、可访问性、响应式）

项目总计提供**12大功能模块**，包含**60+子功能**，为用户提供从错题录入、练习、复习到统计分析的完整学习闭环。
